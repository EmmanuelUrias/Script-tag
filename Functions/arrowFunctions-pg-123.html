Learning JavaScript Arrow Functions

IE11 or earlier versions do not support Arrow functions.

Arrow functions are not suited well to define methods pertaining to objects. These functions lack the hoisting feature. You must define an arrow function before you put it to use in a program. Without the definition, you will see an error in the program. The use of const in arrow functions is considered a safer method instead of using the var keyword. This is because a function expression always has a constant value. You should remove the curly braces and the return keyword if only the function consists of a single statement.

const obj = {
name: "Alice",
greet: () => {
console.log(`Hello, my name is ${this.name}`);
}
};

obj.greet(); // Output: "Hello, my name is undefined"

The arrow function doesn't have the 'this' value. If we used a regular function it would display the name in the console.log()
